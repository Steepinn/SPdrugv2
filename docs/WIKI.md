## Требования и установка

- Сервер: **Paper** (API **1.21**, в проекте указан `paper-api:1.21.1-R0.1-SNAPSHOT`).
- **Java 21** (см. `build.gradle`).
- Сборка: из корня проекта `gradlew build`, готовый jar: `build/libs/SPdrug-<версия>.jar`.
- Положить jar в `plugins/`, перезапустить сервер или использовать менеджер плагинов.
- При первом запуске создаётся `plugins/SPdrug/config.yml`. После правок: `/spdrug reload` (нужны права админа плагина).

## Права

| Право           | Назначение |
|-----------------|------------|
| `spdrug.admin`  | Выдача предметов, `/spdrug reload` (по умолчанию только OP) |
| `spdrug.book`   | Получить книгу: `/spdrug book` или `/spdrug give book` (по умолчанию **у всех**, `default: true`) |

## Команды

Все подкоманды `/spdrug` требуют `spdrug.admin`. Большинство `give` работает **только у игрока** (предметы в инвентарь; не поместившееся падает на землю).

| Команда | Описание |
|---------|----------|
| `/spdrug reload` | Перечитать `config.yml`, перезагрузить рецепты химии и препаратов, перерегистрировать крафты. |
| `/spdrug book` | Выдать книгу рецептов (нужно `spdrug.book` или `spdrug.admin`). |
| `/spdrug give lab` | Установщик лаборатории. |
| `/spdrug give farm` | Установщик теплицы (компостер + зона грядки). |
| `/spdrug give book` | То же, что книга (нужно `spdrug.book` или админ). |
| `/spdrug give seeds` | Все особые семена (каннабис, кока, табак). |
| `/spdrug give raws` | Все типы сырья из `items.herb-raw-types`. |
| `/spdrug give dried` | Все типы сушёного сбора из `items.dried-batch-types`. |
| `/spdrug give chemicals` | Все хим-компоненты из секции `chemicals`. |
| `/spdrug give chemical <id>` | Один хим-компонент по id (например `hc`, `me`). |
| `/spdrug give drugs` | Все финальные препараты из `drugs`. |
| `/spdrug give drug <id>` | Один препарат по id (например `meth`, `cocaine`). |
| `/spdrug give all` | Полный набор: установки, книга, пустая сигарета, семена, сырьё, сушка, химия, препараты. |

Подсказка в игре: `/spdrug` без аргументов (игроку с книгой — строка про `/spdrug book`, админу — полный список).

Подробно по рецептам препаратов и эффектам: файл `docs/PREPARATION.html` (открыть в браузере).

## Предметы и блоки

Настраиваются в `config.yml`: `blocks.*`, `item-materials.*`, тексты в `items.*`.

- **Лаборатория** — предмет-установщик; ПКМ по твёрдому блоку ставит блок лаборатории (по умолчанию `SMOKER`). Открывается GUI синтеза.
- **Теплица** — предмет-установщик; ставит **компостер** как центр зоны грядки (`farm.horizontal-radius`, `farm.vertical-range`).
- **Агрочасы** — ПКМ по выросшему полевому растению (папоротник): информация о прогрессе и бусте.
- **Сырьё / сушёный сбор** — предметы с метками плагина; материал по умолчанию сырья `DRIED_KELP`, сушки `NETHER_WART` (важно согласовать с `recipe` у препаратов с `first-slot-must-be-dried-batch`).
- **Препарат** — ПКМ для употребления (эффекты, титул, боссбар, шанс плохой реакции, передоз при частом приёме).
- **Сигарета** — пустая крафтится из бумаги; заряжается табачным продуктом; загруженная курится ПКМ (лёгкий **реген** на время `cigarette.effect-ticks`).

## Игровой цикл

Упрощённая цепочка:

1. Добыть **особые семена** (трава / геймплей сервера) или выдать админ-командой.
2. Посадить на **грядку** (`FARMLAND`) — растёт **папоротник**; над ним на **+2 по Y** блок **светокамня** ускоряет рост и даёт бонус к урожаю.
3. Сломать созревшее растение → **сырьё** нужного типа (`cannabis` / `cocaine` / `tobacco`).
4. В зоне теплицы: ПКМ по компостеру с сырьём в руке → **сушка** (мини-игра, BossBar).
5. Полученный **сушёный сбор** + цепочка **химии** в **лаборатории** → финальный **препарат** (или промежуточные компоненты).

Параллельно: пшеница в зоне теплицы при полном созревании даёт сырьё типа **каннабис** (базовая ветка фермы — см. `SPdrugListener`).

## Лаборатория

- Реагенты кладутся **слева направо** в слоты последовательности; порядок должен совпадать с рецептом.
- Сопоставление рецепта идёт по **`recipe-keys`**: префиксы `CHEM:<id>` (предмет с меткой химии) и `MAT:MATERIAL` (обычный предмет с таким типом).
- Если у препарата `first-slot-must-be-dried-batch: true`, первый слот должен быть **именно сушёным сбором плагина**, а не обычным блоком/предметом того же материала. Опционально `required-dried-type` (например `cocaine`, `tobacco`) — тип в PDC `dried_batch:<тип>`.
- Поле **`recipe`** — список материалов Bukkit для проверки типа предмета в слоте; длина и порядок должны совпадать с фактической раскладкой.
- Синтез после старта: удержание **стабильности** (ЛКМ по лаборатории, радиус, BossBar). Провал или неверная формула могут **уничтожить лабораторию** (взрыв без разрушения блоков мира — настраивается `laboratory.explosion-*`).
- За один запуск можно положить **несколько полных наборов** — выход умножается, остатки возвращаются.

## Теплица и сушка

- Зона: квадрат вокруг компостера, размер из `farm.horizontal-radius` (например 2 → сторона **5**).
- Сушка: сырьё в основной или второй руке, ПКМ по компостеру; таймер и QTE настраиваются сообщениями в `messages.*` и `farm.drying-duration-ticks`.

---

## Полевые культуры (семена)

- Посадка **только на грядку** (`FARMLAND` / вспаханная земля); иначе сообщение `messages.field-herb-seed-need-farmland`.
- Растение можно **сломать до созревания**: выпадает **1** особое **семя** того же типа (сырьё не даётся). **Созревшее** — **1** или **2** сырья при бусте светокамнем.
- Отдельных предметов «Конопля» / «Сушёный сбор» без типа нет: пшеничное сырьё в теплице и прочие запасные пути дают **каннабис** как базовый тип; `/spdrug give raws|dried` выдаёт только ветки из `items.herb-raw-types` / `items.dried-batch-types`.
- Базовое время созревания: `farm.field-herb-grow-duration-seconds`.
- Ускорение: `farm.field-herb-grow-speed-multiplier` — чем **больше**, тем **быстрее** (время делится на множитель).
- Светокамень на **Y+2** от блока посадки: ускорение и (при сборе в зрелости) больший выход.

---

## Препараты и сигареты

- Каждый препарат в `drugs`: `positive-effects` / `negative-effects`, `bad-trip-chance`, опционально передоз в `drug-use.overdose`.
- `drug-use`: титул, боссбар, урон при «плохой реакции» (`bad-trip-damage-min/max`), вспышки слепоты (`screen-flash-count`, `screen-flash-interval-ticks`, `0` = выкл).
- Сигарета: `cigarette.max-uses`, `cigarette.effect-ticks` — длительность **REGENERATION I** за затяжку.

## Конфигурация (обзор секций)

| Секция | Назначение |
|--------|------------|
| `general` | Радиус BossBar, **`language`**: по умолчанию **`en`**; для русского интерфейса — **`ru`** (`lang/ru.yml` + `/spdrug reload`). |
| `blocks` | Какой блок ставится для лаборатории и фермы. |
| `item-materials` | Материалы предметов плагина (сырьё, сушка, сигарета и т.д.). |
| `farm` | Радиус теплицы, сушка, рост полевых культур; шанс особого семена с травы по умолчанию **10%** (`grass-break-special-seed-chance-percent`). |
| `laboratory` | Длительность синтеза, параметры взрыва. |
| `crafting` | Ингредиенты безформенного крафта установок и предметов. |
| `messages`, `gui`, `bossbar`, `items` | Тексты и материалы GUI. |
| `drug-use` | Эффект употребления, плохая реакция, передоз. |
| `cigarette` | Затяжки и тики эффекта. |
| `commands` | Сообщения команд. |
| `chemicals` | Id химии, отображаемое имя, `recipe` для лаборатории. |
| `drugs` | Финальные препараты: `recipe`, `recipe-keys`, эффекты. |

Цвета в строках: префикс `&` как в классических кодах Minecraft.

## Частые проблемы

**Лаборатория взорвалась после правильных на вид ингредиентов**

- Проверь **`recipe-keys`** и **`recipe`** одновременно: длина, порядок, первый слот — **сушёный сбор** с меткой плагина, не обычный `NETHER_WART`.
- Для кокаина и табака нужен **`required-dried-type`**, совпадающий с типом сушки (`cocaine`, `tobacco`).
- Химия должна быть **синтезирована** (предмет с PDC), а не просто снежок.

**После обновления jar странное поведение**

- Сравни свой `plugins/SPdrug/config.yml` с актуальным из jar / репозитория: новые ключи (`field-herb-*`, `drug-use.bad-trip-*`, и т.д.) могут отсутствовать — тогда подставятся дефолты из кода, но лучше синхронизировать файл.

**Книга рецептов устарела**

- Книга создаётся при выдаче; после смены конфига выдай новую (`/spdrug give book`) или возьми из `give all`.
